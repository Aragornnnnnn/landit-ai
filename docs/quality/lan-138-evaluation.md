# LAN-138 AI 응답 품질 평가

## 평가 기준

- 마무리 멘트는 마지막 사용자 발화와 상대 역할, 구체적인 상황을 반영한다.
- 마무리 멘트는 새 질문이나 대화 종료를 선언하는 메타 표현을 포함하지 않는다.
- `GOOD`은 의미가 명확하고 평가 컨텍스트의 의도를 충족하며 실제 수정이 필요하지 않은 발화다.
- `NEEDS_IMPROVEMENT`는 의미, 문법, 맥락 적합성 또는 역할에 실제 영향을 주는 문제와 구체적 수정 표현이 있을 때만 사용한다.
- 더 세련된 표현이 가능하다는 이유만으로 `NEEDS_IMPROVEMENT`로 판정하지 않는다.

## 수정 전 기준선

- 모델. `openai/gpt-5.4-mini`.
- 사례. 당시 `tests/fixtures/lan_138_quality_cases.json`에 있던 비식별화한 9개 사례.
- 반복 횟수. 사례별 3회.
- 마무리 멘트. 3개 사례 9회에서 새 질문과 메타 종료 문구는 0건이었다. 제보된 어색함은 현재 사례로 재현하지 못했다.
- 메시지 피드백. 처음 5개 사례 15회는 기대 레이블과 모두 일치했다. 직설적인 구어체 경계 사례를 추가한 뒤 6개 사례 18회를 다시 실행했을 때 `feedback-direct-colloquial-boundary`가 3회 모두 `NEEDS_IMPROVEMENT`였다.
- 원인. `_message_feedback_judgement_policy()`와 AI_MESSAGE 예시가 `Why do you wanna know that?`를 방어적이거나 따지는 표현으로 고정해 `NEEDS_IMPROVEMENT`로 유도했다.

## 수정 후 검증

- 구어체가 명확하고 평가 컨텍스트에 맞으면, 더 부드러운 대안이 있더라도 `GOOD`으로 우선 판단하도록 정책과 예시를 수정했다. 특정 문장을 그대로 정답 예시에 넣지 않고 같은 의도의 다른 표현을 사용해 예시 복사를 줄였다.
- `feedback-direct-colloquial-boundary`의 AI 발화에서 이유 설명을 제거해, 사용자의 이유 질문이 자연스러운 상황임을 명확하게 만들었다. 기존 endpoint mock도 같은 조건과 `GOOD` 결과로 맞췄다.
- 사용자 제보 문구와 프롬프트를 대조한 결과, 기존 예시가 `Let's wrap up here`, `Let's pause here`, `여기서 마무리하자`를 정답으로 직접 제시하고 있었다. 메타 종료 예시를 상황별 반응으로 교체하고, 같은 유형의 모델 응답은 `AI_RESPONSE_INVALID` 502로 거절하도록 했다.
- 메타 종료 판정은 스마트 따옴표와 영문·한글 변형을 정규화해 검사한다. `wrap up the gifts`처럼 대화 종료가 아닌 표현은 거절하지 않는다.
- 품질 평가 도구는 운영 검증을 통과한 응답만 보지 않고 LLM 원본 후보를 검사한다. 결과 파일에는 실행 시각, 모델명, 사례 파일 경로와 SHA-256, 반복 횟수, 평가 종류를 함께 기록한다.
- 최종 평가는 2026-07-14 15:30:44 KST에 `openai/gpt-5.4-mini`로 실행했다. 사례 파일 SHA-256은 `6f606e5c9d8fc71965ee4c2cce1dc2804116d4999b9d7518232400b89780ee61`이다.
- 마무리 4개 사례를 3회씩 실행한 12회에서 새 질문, 메타 종료 문구, 기대 맥락 불일치가 모두 0건이었다. `MAX_TURNS_REACHED` 사례도 여권과 여행 맥락을 유지했다.
- 메시지 피드백 6개 사례를 3회씩 실행한 18회에서 기대 레이블 불일치는 0건이었다. 직설적인 구어체, 짧은 답변, USER First 시작 발화는 `GOOD`을 유지했고, 문법 오류와 맥락 이탈은 `NEEDS_IMPROVEMENT`를 유지했다.
- 맥락 적합성은 의미 기반 자동 검증으로 강제하지 않는다. 정상적인 동의어·간접 표현을 잘못 거절할 위험이 있으므로, 실제 제보가 추가되면 비식별화한 전체 입력을 사례로 추가해 반복 검증한다.

## GOOD 과보정 점검

- 품질 평가 결과에는 기대 피드백 레이블도 기록해, `GOOD` 오판과 `NEEDS_IMPROVEMENT` 오판을 각각 집계할 수 있게 했다.
- 문법은 맞지만 AI 질문과 무관한 답변, 교수에게 무례한 거절 사례를 추가했다. 메시지 피드백 suite는 GOOD 4개, NEEDS_IMPROVEMENT 4개로 균형을 맞췄다.
- 2026-07-14 15:45 KST에 `openai/gpt-5.4-mini`로 8개 사례를 각 5회 실행했다. 기대 GOOD 20회는 모두 GOOD, 기대 NEEDS_IMPROVEMENT 20회는 모두 NEEDS_IMPROVEMENT로 판정되어 두 방향의 오판이 모두 0건이었다.
- 이 결과는 고정된 비식별화 사례에서의 일관성만 보장한다. 실사용 문장과 역할 조합을 계속 수집해 같은 suite에 추가한다.
